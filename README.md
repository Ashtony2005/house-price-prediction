House Price Prediction

Internship Week 1 project — a regression model that predicts house prices from property
features and identifies which features influence price the most.

Overview

Real estate buyers and sellers often rely on guesswork or outdated comparisons to judge a
property's fair value. This project builds and compares two regression models on a real
housing dataset to estimate price from structural features (area, rooms, amenities), and
digs into why the model predicts what it predicts — not just how accurate it is.

Dataset

Kaggle — Housing Prices Dataset (yasserh)

545 property listings, 13 columns: price, area, bedrooms, bathrooms, stories,
mainroad, guestroom, basement, hotwaterheating, airconditioning, parking,
prefarea, furnishingstatus. No missing values, no duplicate rows.

Approach


Explore the raw data — shape, dtypes, missing values, target vs. features
Clean — map yes/no columns to 1/0, one-hot encode furnishingstatus
Model — 80/20 train/test split; Linear Regression vs. Random Forest Regressor,
evaluated on MAE, RMSE, and R²
Visualize — price distribution, correlation heatmap, actual-vs-predicted scatter,
plus bonus feature-importance and residual plots
Interpret — which features matter most, how accurate the model really is, and one
actionable recommendation for a real estate business

A non-obvious finding

The raw CSV is sorted by price (descending). This is invisible during a normal
train_test_split (which shuffles by default), but it silently breaks unshuffled
k-fold cross-validation — each fold ends up containing only a narrow price band, so R²
scores come out wildly negative (around -10) even though the model itself is fine. Always
check whether your data is sorted before trusting an unshuffled CV score.
